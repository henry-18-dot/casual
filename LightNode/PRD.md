# LightNode PRD (产品需求文档)

版本: v0.1
日期: 2026-03-16
负责人: 产品与研发协作

## 1. 产品愿景
LightNode 旨在把冗长的 AI 对话压缩为极简、图形化、可交互的知识脉络，帮助学习者以最低认知负载复习知识。它解决的是“对话后即遗忘、记录太长翻不动”的知识沉淀难题。

## 2. 目标用户
- 年龄范围: 初中生至 30 岁左右的年轻学习者
- 用户画像: 习惯使用 AI 辅助学习的终身学习者
- 典型场景: 课内知识复习、考试备考、技能学习、主题研究

## 3. 核心价值
- 无痛导入: 复制粘贴或链接导入即可生成
- 视觉导航: 像地图一样浏览知识
- 分层记忆: 极简概述与详细论述分层呈现

## 4. 成功标准
- 次日留存率: 35% (口径: 打开过应用即计入)
- 复习频率: 平均每周 3 次以上 (口径: 打开过任意创建超过 24 小时的旧项目计一次)

## 5. 范围定义
### 5.1 In Scope (第一版)
- 桌面应用: macOS 优先，同时发布 Windows
- 免安装绿色版分发
- 多渠道导入: 分享链接解析、官方导出文件 (JSON/MD)、直接粘贴对话文本
- AI 消化: 关键词提取、两层摘要、节点结构化 JSON
- 动态思维导图: Canvas 或 SVG 渲染，缩放与动画
- 离线查看: 第一击内容离线可用
- 搜索定位: 标题与正文关键词匹配
- 手工编辑: 新增节点、编辑节点、拖拽调整层级
- 动态重构: 新内容导入后提示重构，支持 Accept
- 数据本地存储与加密
- 账户体系: 本地匿名优先，可选登录
- 付费策略: 月订阅，含试用或新用户额度

### 5.2 Out of Scope (第一版明确不做)
- 协同编辑
- 多设备同步
- 导出 PDF/图片
- 社交分享

## 6. 用户故事
1. 作为学习者，我希望粘贴一段 AI 对话或链接，在 3 分钟内看到动态思维导图，以便快速复习。
2. 作为学习者，我希望点击节点先看到极简概述，再点击看到详细论述，以便分层记忆。
3. 作为学习者，我希望 AI 提示分类建议但保留最终决定权，以便保持自己的结构偏好。
4. 作为学习者，我希望导图在离线时仍能查看节点标题与极简概述，以便通勤或断网复习。
5. 作为学习者，我希望导图可手工编辑且撤销可用，以便修正 AI 输出。
6. 作为学习者，我希望在 AI 低置信度时得到提醒，以便校验内容。
7. 作为学习者，我希望导入失败时也能保存原文草稿，以便稍后补处理。

## 7. 核心流程
1. 导入
- 用户选择当前主文件夹或新建主文件夹
- 通过分享链接、JSON/MD 文件、或粘贴文本导入
- 导入后进入解析队列

2. AI 消化
- 文本结构化 JSON 解析
- 生成关键术语与两层摘要
- 生成节点树与关联信息

3. 展示与复习
- 首次进入展示动画: 模块淡出 -> 标题归位 -> 视野拉远 -> 导图展开
- 节点点击: 第一击弹出极简概述，第二击展开详细论述与图形内容

4. 动态重构
- 新内容导入后触发结构建议
- 差异高亮预览: 旧节点变灰，新节点高亮
- 用户 Accept 后平滑过渡

## 8. 功能规格
### 8.1 导入与解析
- 分享链接: 解析公开分享链接内容 (需用户确认授权)
- 文件导入: 支持 JSON 与 MD
- 文本粘贴: 仅保留文本逻辑，舍弃时间戳与冗余角色信息，保留代码块与公式
- 重复导入: 合并更新并高亮新增分支
- 失败路径: 原文存入“待处理”侧边栏

### 8.2 AI 消化与总结
- 极简概述: 小于 30 字 (中文) 或 30 词 (英文)
- 详细论述: 100-300 字 (中文) 或 100-300 词 (英文)
- 必含字段: 核心标题、关键定义、高亮重点、推导逻辑或流程、关联公式
- 置信度 < 0.7: 节点显示黄色问号提示

### 8.3 节点与导图渲染
- 节点内容类型: 纯文本、LaTeX 公式、流程图、Mermaid、代码块
- Mermaid 与公式必须可渲染，离线使用本地渲染库
- 深度超过 5 层时视觉弱化

### 8.4 符号化反馈
- 提供“详尽一点”和“简短一点”图标
- 无确认锁定功能，避免锁死节点内容

### 8.5 编辑与撤销
- 支持新增节点、编辑内容、重命名、拖拽层级
- 撤销范围: 删除节点、移动分支、改名、内容编辑、接受 AI 重构建议
- 用户手工修改过的节点不再被 AI 自动重构

### 8.6 搜索与导航
- 关键词搜索: 标题与正文文本匹配
- 聚焦模式: 只看某一分支
- 无小地图

### 8.7 账户与计费
- 本地匿名可用
- 轻量账户: 手机号或邮箱登录
- 订阅制: 月付，含试用或新用户额度

### 8.8 设置
- 动效开关: 可关闭其他动态效果
- 原始对话片段保存长度可调
- 关闭保留后立即清除缓存

## 9. 数据模型 (核心实体)
### 9.1 Project
- id
- name (1-24 字)
- created_at
- updated_at

### 9.2 MainFolder
- id
- project_id
- name
- order

### 9.3 Node
- id
- parent_id
- title
- summary_short
- summary_long
- type
- raw_ref
- confidence
- edited_by_user (bool)

### 9.4 RawRef
- id
- node_id

- source_type (link, file, paste)

### 9.5 ImportTask
- id
- source

### 9.6 UserPrefs

## 10. 离线与在线
- 离线可用: 节点标题、极简概述、Mermaid/公式渲染资源
- 在线必需: 导入、第二击详细论述、AI 总结与动态重构

## 11. 错误处理
- 导入失败: 以拟人化提示告知，可存入“待处理”草稿
- AI 解析失败: 自动重试 1 次，失败后显示“手动重试”按钮
- 导图生成失败: 降级为大纲式极简笔记

## 12. 性能与限制
- 单次解析上限: 10 万 token
- 目标等待时长: 20-30 秒
- 解析策略: 分段处理并合并

## 13. 视觉与交互规范
- VS Code 风格深色工业配色
- 项目卡片: 圆角矩形、粗边框
- 左侧黑白灰几何圆圈标识，右侧 2-6 字项目描述
- 丝滑转场: 模块淡出 -> 标题归位 -> 视野拉远 -> 导图展开

## 14. 安全与隐私
- 本地数据库加密 (SQLCipher 或等效方案)
- 用户数据默认不用于模型优化
- 参与改进计划需用户勾选，提供额外免费额度

## 15. 风险与依赖
- 分享链接可解析性依赖第三方平台策略
- 10 万 token 解析的成本与时延需控制
- 动画与性能需通过懒加载与渐进渲染平衡
