MorphLex progress log
====================

Project status: planning complete, implementation not started
Version target: v0.1 desktop application (Windows-first, local-first)

----------------------------------------
1. What is already locked
----------------------------------------

Product identity
- Working name locked: MorphLex
- Core definition locked: local-first morphology visualization vocabulary tool
- Product scope locked for v0.1

Platform and architecture
- Desktop app only
- Tauri + React + Vite + TypeScript + SQLite
- Windows-first
- Single main window
- Single routed main content area
- No multi-window workflow

Core data model
- One import action = one session
- One session may contain multiple batches
- Each batch max 120 words
- Global word table deduplicated
- Session/batch keep relational links
- Morpheme categories: prefix / root / suffix
- Word may map to multiple morphemes
- Unrecognized words must be preserved and marked

Feature scope locked in MVP
- Guest mode only
- Account panel UI only
- Phone binding placeholder only
- Cloud sync placeholder only
- Local avatar upload
- Local settings persistence
- Batch import by text / txt / csv
- Over-limit split confirmation
- Import history
- Session detail
- Session-scoped search
- Re-analyze current session
- Delete current session
- Morpheme collection pages
- Word detail modal
- Root-only generate-more
- AI provider abstraction reserved
- Local mock analysis only

Out of scope locked
- Real login
- Real SMS
- Real cloud sync
- Multi-device sync
- Export
- Word-level deletion
- Manual editing of analysis results
- Manual append to existing session
- Mobile adaptation
- Spaced repetition / review engine
- Check-in / streak system
- Collaboration / sharing
- Online-dependent AI analysis
